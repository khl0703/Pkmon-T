# CLAUDE.md — 포켓몬스터 토론토 프로젝트 가이드

## 프로젝트 개요

**포켓몬스터 토론토**는 캐나다 토론토를 배경으로 한 2D 탑다운 포켓몬 팬게임이다.
**모티브**: 닌텐도 NDS 포켓몬 게임들 (HGSS, DP/Pt, BW, BW2). 본가 NDS 게임이 기준.
HTML5 Canvas + Vanilla JavaScript로 구현한다.
데모 범위: 로바츠 도서관(1번째 체육관) → University Ave → 퀸즈파크(2번째 체육관).

---

## NDS 포켓몬 레퍼런스 (개발 필수 참고)

모든 UI, 조작감, 비주얼은 NDS 포켓몬 본가(Gen4: HGSS/DPPt, Gen5: BW/BW2) 기준.

### 화면 / 해상도 / 타일

- NDS 화면: **256×192 픽셀**. 우리도 동일 내부 해상도 사용
- **맵 타일**: 16×16px. 화면에 16×12 타일 그리드
- **오버월드 캐릭터**: 16×32px (4방향 × 3프레임 걷기)
- **배틀 스프라이트**: Gen5 기준 96×96px
- **아이콘 (파티)**: 32×32px

### 버튼별 기능 — 오버월드

| NDS 버튼 | 키보드 | 기능 |
|---------|--------|------|

### 배틀 UI 레이아웃

```
[상대 이름] ♂ Lv.XX
████████░░░░  HP바

       (상대 스프라이트)

  (내 스프라이트 뒷면)

           [내 이름] ♀ Lv.XX
           ████████░░░░  HP바
           HP: 120/150
           EXP: ████░░░░
─────────────────────────
│ 싸우다  │  가방    │
│ 포켓몬  │  도망    │  ← 2×2 그리드
```

- HP바 색상: 초록(>50%) → 노랑(20~50%) → 빨강(<20%)
- 경험치바: 파란색
- "싸우다" → 기술 4개 (2×2), 각각 PP 표시 + 타입 색상 배경
- 기술 상세: 타입, 분류(물리/특수/변화), 위력 표시

### 인게임 메뉴 (X/S키)

1. **포켓몬** — 파티 6마리, HP바, 상태이상 아이콘
2. **가방** — 카테고리: 도구/회복/포켓볼/기술머신/열쇠
3. **트레이너카드** — 이름, 배지, 플레이시간
4. **세이브** — 저장
5. **옵션** — 설정
6. **도감** — 잡은/본 포켓몬

### 옵션 (본가 + 우리 게임)

**본가 표준**: 문자속도(느림/보통/빠름), 배틀연출(보기/안보기), 배틀스타일(바꿀래?/안바꿈), 사운드(스테레오/모노)
**우리 추가**: 언어(KR/EN), BGM볼륨, SFX볼륨, 게임속도(x1/x2/x3)

### 대화창

- 화면 하단, 높이 ~48px, 전체 너비
- 텍스트 한 글자씩 타이핑 출력 (문자속도 설정에 따라)
- A버튼(X키)으로 다음 진행
- ▼ 바운스 화살표로 "다음" 표시
- 선택지는 별도 작은 윈도우 (예: 예/아니오)

### 화면 전환 효과

- **맵 이동**: 페이드 아웃(검정) → 페이드 인
- **배틀 진입**: 깜빡 → 세로줄무늬 와이프 또는 스파이럴 → 배틀화면
- **배틀 종료**: 페이드 아웃 → 필드 복귀
- **건물/층 이동**: 빠른 암전 (0.3초)

### 사운드 설계

- 각 맵/지역 고유 BGM. 건물 내부/외부 다름
- 배틀 BGM: 야생전/트레이너전/관장전/라이벌전 각각 다름
- SE: 확인음, 취소음, 커서이동음, 데미지음, HP경고 비프, 포켓볼 던지기/잡기, 레벨업 팡파레
- HP <20%: "삐삐삐" 반복 경고음

## 기술 스택

- **순수 HTML5 + CSS + JavaScript (단일 index.html 파일)**
- **Canvas 2D API** — 모든 게임 화면을 Canvas에 렌더링
- **도트(픽셀아트) 기반** — NDS 스타일 내부 해상도 256×192, CSS로 640×480 스케일링
- **image-rendering: pixelated** — 픽셀이 선명하게 보이도록
- **외부 라이브러리 없음** — Vanilla JS only
- **데이터**: 포켓몬/맵/트레이너 데이터는 JS 객체 또는 JSON
- **저장**: localStorage 사용
- **폰트**: Google Fonts — 'DotGothic16' (한국어 도트), 'Press Start 2P' (영문 도트)
- **아트 스타일**: 클래식 포켓몬 GBA/NDS 느낌의 2D 도트 그래픽. 모든 UI는 RPG 윈도우 박스 스타일

## UI 스타일 규칙

- 모든 UI 박스는 `drawWin(x,y,w,h)` 함수로 그린다 (RPG 윈도우 스타일: 어두운 테두리 + 밝은 내부 + 하이라이트/그림자)
- 커서는 `▶` 문자를 빨간색으로 표시, 바운스 애니메이션 포함
- 내부 해상도는 256×192 (NDS 스타일). 모든 좌표/크기는 이 해상도 기준
- `ctx.imageSmoothingEnabled = false` 필수 — 픽셀이 흐려지면 안 됨

## 코딩 규칙

1. **모든 코드는 index.html 하나에 작성한다** (CSS는 `<style>`, JS는 `<script>` 태그 안에).
2. **내부 해상도 256×192 (W=256, H=192)** — Canvas는 이 크기로 그리고 CSS가 640×480으로 확대.
3. **ctx.imageSmoothingEnabled = false** — 매 프레임 시작 시 반드시 설정.
4. **도트 폰트만 사용**: 한국어 = 'DotGothic16', 영문 = 'Press Start 2P'. 안티앨리어싱 폰트 사용 금지.
5. **게임 상태는 전역 객체로 관리**: `GameSettings`, `PlayerData` 등.
6. **화면 전환은 `screen` 변수**로: `'title'` → `'prologue'` → `'field'` → `'battle'` 등.
7. **gameLoop()** 함수가 매 프레임 `render()` 호출, `screen`에 따라 분기.
8. **i18n**: 모든 텍스트는 `t('key')` 함수. `i18n` 객체에 ko/en 키 추가.
9. **입력**: `keys` (누르고 있는 키), `ck(key)` (한 번만 인식).
10. **모든 그래픽은 Canvas API로 직접 픽셀 렌더링** — 이미지 파일 없이 코드로 도트를 찍는다. 추후 스프라이트시트 도입 가능.

## NDS 포켓몬 게임 레퍼런스 (HGSS / DPPt / BW / BW2)

이 게임은 닌텐도 DS 포켓몬 게임들 (4세대~5세대)을 모티브로 한다. 아래는 구현 시 참고해야 할 모든 시스템 상세.

### NDS 하드웨어 기본 사항

- **해상도**: 각 화면 256×192 (상하 듀얼 스크린)
- **우리 게임**: 싱글 스크린 256×192로 구현, CSS로 확대
- **색상**: 262,144색 (18-bit). 실제로는 팔레트 기반으로 제한된 색상 사용
- **타일 크기**: 16×16 픽셀 (필드 맵 기본 단위)
- **캐릭터 스프라이트**: 보통 16×32 (1타일 너비, 2타일 높이) 또는 32×32

### NDS 포켓몬 버튼별 역할 (실제 DS 하드웨어 기준)

| DS 버튼 | 필드 | 배틀 | 메뉴/UI | 대화 |
|---------|------|------|---------|------|
| **D-Pad (십자키)** | 4방향 이동 | 기술 선택 커서 | 항목 선택 | — |
| **A 버튼** | 대화/조사/확인 | 확인 선택 | 확인 | 다음 텍스트 |
| **B 버튼** | 달리기(Running Shoes) | 취소/뒤로 | 취소/닫기 | 텍스트 스킵(빨리넘기기) |
| **X 버튼** | 메뉴 열기 (Gen4부터) | — | — | — |
| **Y 버튼** | 등록 아이템 사용 | — | 정렬/보조기능 | — |
| **L 버튼** | 페이지 넘기기(좌) | 포켓몬 교체(BW) | 페이지 넘기기 | — |
| **R 버튼** | 페이지 넘기기(우) | 포켓몬 교체(BW) | 페이지 넘기기 | — |

**주요 변화:**
- **DPPt**: X버튼으로 메뉴 열기 도입. Y버튼에 아이템 등록. B+방향키=달리기
- **HGSS**: 터치스크린에 메뉴 항상 표시. 따라가는 포켓몬 시스템. B=달리기(토글/홀드 옵션 제공)
- **BW/BW2**: 메뉴 속도 대폭 개선. L/R로 배틀 중 포켓몬 교체 가능. X로 빨간 X표시(메뉴 바로 닫기)

### 메인 메뉴 항목 (X 버튼으로 열기)

Gen4 (DPPt/HGSS) 기본 메뉴:
1. **포켓몬 (Pokémon)** — 파티 6마리 보기. 순서 변경, 기술 확인, 아이템 사용
2. **가방 (Bag)** — 아이템 카테고리별 정리: 일반아이템/약/포켓볼/기술머신/열쇠도구/나무열매
3. **트레이너 카드 (이름)** — 배지, 플레이 시간, 도감 수, ID
4. **세이브 (Save)** — 저장
5. **옵션 (Options)** — 텍스트 속도, 배틀 장면, 배틀 스타일, 사운드
6. **도감 (Pokédex)** — 획득 후 표시

Gen5 (BW/BW2) 메뉴 (2열 그리드):
- 포켓몬 / 도감 / 가방 / 트레이너카드 / 세이브 / 옵션
- BW2에서 추가: 메달, C-Gear 등

### 배틀 시스템 상세

#### 배틀 화면 구성 (NDS)
```
┌─────────────────────────────┐
│ [상대 포켓몬 스프라이트]      │  ← 상단: 상대
│ ┌──────────────┐             │
│ │ 이름  Lv.XX  │             │  ← 상대 HP바 (좌상단)
│ │ ████████░░░░ │             │
│ └──────────────┘             │
│                              │
│        [내 포켓몬 스프라이트]  │  ← 하단: 내 포켓몬
│              ┌──────────────┐│
│              │ 이름  Lv.XX  ││  ← 내 HP바 (우하단)
│              │ ████████░░░░ ││
│              │ HP: 45/78    ││  ← 숫자 HP는 내 것만 표시
│              └──────────────┘│
│ ┌──────────────────────────┐ │
│ │ 싸우다  가방  포켓몬  도망 │ │  ← 4개 커맨드
│ └──────────────────────────┘ │
└─────────────────────────────┘
```

#### 배틀 커맨드 4개
1. **싸우다 (Fight)** → 기술 4개 선택 화면으로
2. **가방 (Bag)** → 아이템 사용 (상처약, 포켓볼 등)
3. **포켓몬 (Pokémon)** → 교체
4. **도망 (Run)** → 야생 배틀에서만 가능. 트레이너전에서는 불가

#### 기술 선택 화면
```
┌──────────┬──────────┐
│ 기술1    │ 기술2    │  ← 2×2 그리드
├──────────┼──────────┤
│ 기술3    │ 기술4    │
└──────────┴──────────┘
PP: 15/20   타입: 불꽃    ← 선택한 기술의 PP와 타입 표시
```

#### 데미지 공식 (Gen4-5 공통)
```
Damage = ((2×Level/5+2) × Power × A/D) / 50 + 2
       × Modifier

Modifier = STAB × Type × Critical × random(0.85~1.0) × Other

- Level: 공격 포켓몬 레벨
- Power: 기술 위력
- A: 공격측 공격/특공 (물리/특수에 따라)
- D: 방어측 방어/특방
- STAB: 자속보정 1.5 (기술 타입 = 포켓몬 타입)
- Type: 타입 상성 (0, 0.25, 0.5, 1, 2, 4)
- Critical: 급소 시 1.5 (Gen6부터, Gen4-5는 2.0)
- random: 0.85~1.00 사이 랜덤 계수
```

#### 야생 포켓몬 인카운터 시스템
- **풀숲 타일**: 걸을 때마다 인카운터 확률 체크
- **기본 인카운터율**: 매 스텝 약 8.5% (187/2048) — DPPt 기준
- **인카운터율 영향 요소**: 자전거(감소), 스프레이(레벨 이하 차단), 특성(깨어남주의/벌레의알림 등)
- **포켓몬 결정**: 출현 테이블에서 확률 가중치로 선택 → 레벨 범위 내 랜덤
- **인카운터 연출**: 화면 플래시 + 배틀 전환 이펙트 (화면이 깨지듯 조각남 — BW 스타일)

#### 포켓몬 잡기 공식 (Gen4-5)
```
a = (3×maxHP - 2×curHP) × catchRate × ballBonus / (3×maxHP) × statusBonus

statusBonus: 잠듦/얼음 = 2.5, 마비/독/화상 = 1.5, 없음 = 1
ballBonus: 몬스터볼=1, 슈퍼볼=1.5, 하이퍼볼=2, 마스터볼=255

b = 65536 / √(√(255×3÷a))

4번 체크: 각각 random(0~65535) < b 이면 성공
4번 모두 성공 → 잡기 성공
실패 횟수에 따라 흔들림 횟수 결정 (0~3번)
```

### 필드 맵 시스템

#### 타일 구조
- **타일 크기**: 16×16 픽셀 (NDS 표준)
- **맵 레이어**: 최소 3개
  - Layer 0: 바닥 (잔디, 도로, 물)
  - Layer 1: 장식 (꽃, 돌, 가구)
  - Layer 2: 위에 겹치는 것 (지붕, 나뭇잎 — 캐릭터 위로 렌더링)
- **충돌 레이어**: 통과 가능/불가 타일 구분

#### 캐릭터 이동
- **그리드 기반**: 16px 단위로 정확히 이동
- **이동 속도**: 걷기 = 2px/frame, 달리기 = 4px/frame (60fps 기준)
- **이동 애니메이션**: 4프레임 (정지, 왼발, 정지, 오른발) × 4방향 = 16프레임
- **방향 전환**: 제자리에서 방향만 바꿀 수 있음 (첫 입력 = 방향 전환, 유지 = 이동)
- **NPC와 충돌**: 통과 불가, 대화 트리거

#### NPC 시스템
- **고정 NPC**: 제자리에서 특정 방향 바라봄
- **랜덤 이동 NPC**: 1~4칸 범위 랜덤 이동, 일정 간격
- **트레이너 NPC**: 시선 범위(1~5칸) 내에 플레이어 진입 시 → 느낌표(!) → 걸어와서 배틀
- **대화 NPC**: A버튼으로 대화. 대화 종료 후 다시 말 걸면 반복 or 다른 대사

#### 대화창 시스템
- **위치**: 화면 하단 (높이 약 48px, 너비 약 240px)
- **텍스트 표시**: 한 글자씩 타이핑 효과 (빠름/보통/느림)
- **A 버튼**: 다음 텍스트로 / 타이핑 중이면 즉시 전부 표시
- **B 버튼**: 텍스트 빠르게 넘기기 (연타 시 빠르게 진행)
- **선택지**: "예/아니오" 같은 선택은 우측에 윈도우 박스로 표시
- **이름 삽입**: "PLAYER 은/는 상처약을 받았다!" — 변수 치환

#### 포켓몬 센터
- **기능**: 포켓몬 전원 HP/PP/상태이상 전부 회복 (무료)
- **연출**: 간호순이 "포켓몬을 맡겨주세요" → 포켓볼 올려놓는 연출 → "건강해졌습니다!"
- **PC 박스**: 포켓몬 보관함 접근 가능
- **2층**: 거래/대전 (우리 게임에서는 생략 가능)

#### 포켓몬 상점 (Mart)
- **독립 건물** 또는 **센터 내부** (BW부터 센터 통합)
- **구매**: 아이템 목록에서 선택 → 개수 선택 → 금액 확인 → 구매
- **판매**: 가방에서 아이템 선택 → 개수 → 판매가(구매가의 절반)
- **배지에 따라 상품 해금**: 배지 수가 늘면 좋은 아이템 판매

### UI 윈도우 스타일 (NDS)

#### 윈도우 박스 그리기 규칙
- **외곽 테두리**: 2~3px 어두운 색
- **내부 배경**: 밝은 크림/흰색
- **하이라이트**: 좌측+상단 내부에 1px 밝은 선
- **그림자**: 우측+하단 내부에 1px 어두운 선
- **모서리**: NDS에서는 거의 직각 (라운드 없음)

#### HP바 색상 규칙
- **초록 (>50%)**: HP가 절반 이상
- **노랑 (20~50%)**: HP가 1/5 ~ 1/2
- **빨강 (<20%)**: HP가 1/5 미만 → 경고음 재생

#### 텍스트 스타일
- **일반 대화**: 흰색 배경에 검은 텍스트
- **배틀 로그**: 반투명 어두운 배경에 흰색 텍스트 (BW 스타일)
- **시스템 메시지**: "세이브 중입니다..." 등 — 작은 팝업 윈도우

### 경험치 & 레벨업

#### 경험치 획득 공식 (Gen5 — BW/BW2)
```
EXP = (a × b × L) / (5 × s) × ((2L+10)^2.5 / (L+Lp+10)^2.5) + 1

a = 야생 1.0, 트레이너 1.5
b = 상대 포켓몬 기본 경험치
L = 상대 레벨
Lp = 내 포켓몬 레벨
s = 참여 포켓몬 수 (학습장치 등 보정)
```

#### 레벨업 시:
1. 스탯 재계산 (종족값 + 개체값 + 노력치 기반)
2. 새로운 기술 학습 가능 여부 체크 → 4개 초과 시 교체/포기 선택
3. 진화 조건 확인 → 조건 충족 시 진화 연출

### 사운드 & 음악 참고

- **타이틀 BGM**: 장엄하고 서사적 (DPPt 오프닝 스타일)
- **필드 BGM**: 지역마다 다른 테마 (도시=밝고 경쾌, 동굴=긴장, 도서관=조용하고 신비)
- **배틀 BGM**: 야생=빠르고 긴장, 트레이너=더 빠름, 관장=웅장
- **효과음**: 메뉴 커서 이동(틱), 확인(핑), 취소(뿌), HP감소(삐삐), 레벨업(빠빠빰)
- **현재**: 사운드 없이 개발, 추후 웹 오디오 API로 추가

### 화면 전환 효과

- **맵 이동**: 페이드 아웃(검정) → 페이드 인 (0.3~0.5초)
- **배틀 진입**: 화면 플래시(흰색, 3회) → 화면 깨지는 효과(BW) 또는 사선 와이프(DPPt) → 배틀 화면
- **배틀 종료**: 페이드 아웃 → 필드 복귀
- **건물 진입**: 즉시 전환 또는 짧은 페이드
- **체육관 진입**: 특별 전환 효과 (배지 표시 등)

### 타입 상성표 (18타입)

구현 시 2D 배열로 저장: `typeChart[attackType][defendType]` = 배수 (0, 0.5, 1, 2)

핵심 상성 (데모에서 자주 나오는 것):
